# Weekly Fixture

Static web page showing this week's FHC fixtures, styled after the club's social-media graphics.

Views, switched via the tabs at the top:

| Tab | Contents |
| --- | --- |
| **All** | Every fixture for the week |
| **Premier League** | Men's PL / PLR and Women's PL / PLR |
| **Seniors** | All other senior men's / women's grades (Pennant A–D, Metro 1/2, etc.) |
| **Midweek** | All midweek competitions |
| **Juniors** | All junior competitions |

Data comes from `data/fixtures.json`, a rolling 2-week snapshot of the four category Google Calendars. It is regenerated daily by the `sync-calendars` GitHub Action (via `npm run generate-fixtures-json`) and committed back to the repo, which triggers a Netlify redeploy.

## Folder layout

```text
weekly-fixture/
  index.html, subscribe.html, 404.html       # entry points (Netlify publish root)
  css/styles.css                             # shared stylesheet
  js/app.js                                  # fixtures page logic
  js/subscribe.js                            # subscribe page logic
  images/og-image.png                        # social preview (generated)
  images/logos/*.png                         # team logos
  icons/                                     # favicon set + site.webmanifest
  data/subscribe-data.json                   # generated by `npm run generate-docs`
  src/og-card.html                           # source for the og-image generator
```

## Run locally

From the **repo root**:

```bash
npm run dev
```

Then open <http://localhost:3000>.

## Pages

| File | Purpose |
| --- | --- |
| `index.html` | This week's fixtures, grouped by day |
| `subscribe.html` | Friendly alternative to `docs/competitions-mobile.md` — per-category calendar subscribe instructions. Linked from the fixtures page. |

## Subscribe page data

`subscribe.html` reads `weekly-fixture/data/subscribe-data.json`. That file is regenerated automatically whenever you run:

```bash
npm run generate-docs
```

The generator (see [src/setup/generate-docs.js](../src/setup/generate-docs.js) — `generateSubscribeData`) pulls the same category + calendar metadata used for the mobile markdown, so the subscribe page and `docs/competitions-mobile.md` stay in sync.

## How the data is parsed

- `js/app.js` fetches `/data/fixtures.json` — a pre-parsed list of events for the next ~2 weeks across the four category calendars. The file is regenerated daily by the GitHub Action and served as a regular static asset (fully CDN-cached, no runtime Google dependency).
- Only events with `DTSTART` in the current Mon–Sun window are kept.
- Each event's `SUMMARY` is parsed as `"<COMP> - <HOME> vs <AWAY>"`.
- The competition string is classified into one of the four views:
  - `midweek` / `juniors` feeds map straight to their tabs
  - `mens` / `womens` split on whether the comp contains `PL` / `PLR` (→ `pl` tab) or not (→ `club` tab)
- A short grade code is derived for display (`Men PEN A` → `MPA`, `Women M1 NW` → `WM1`, etc.) — see `buildShortCode()` in `js/app.js`.

## Updating

- **Calendar IDs**: category calendar IDs are read from `config/settings.json` (`categoryCalendars.*.calendarId`). If a calendar is recreated, update settings.json and re-run `npm run generate-fixtures-json`.
- **Logos**: drop a new `<ABBR>.png` into `images/logos/`. Abbreviations come from `config/mappings-club-names.json`. Non-PNG extensions can be registered in `LOGO_EXTENSIONS` in `js/app.js`.
- **Styling**: edit `css/styles.css`.
- **OG image**: edit `src/og-card.html`, then run `npm run generate-og-image` from the repo root.

## Notes

- Fixtures are filtered client-side against Melbourne time, so visitors in other timezones still see the correct week.
- There is no runtime call to Google Calendar from the browser — data comes from a committed JSON file refreshed daily by the GitHub Action.
